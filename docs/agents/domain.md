# Domain Docs

This is a single-context repo.

Before exploring, read root `CONTEXT.md` and `docs/adr/` when they exist. If they do not exist, proceed silently.

Use the glossary vocabulary from `CONTEXT.md` in issue titles, PRDs, implementation plans, tests, and code comments. If a needed concept is missing, note that `/domain-modeling` should add it rather than inventing competing names.

Flag any contradiction with existing ADRs explicitly instead of silently overriding prior decisions.
